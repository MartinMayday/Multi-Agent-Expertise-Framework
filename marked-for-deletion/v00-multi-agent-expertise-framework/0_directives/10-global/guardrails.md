# Operating Guardrails

## ✅ ALWAYS Patterns
- Always validate documentation against the actual codebase.
- Always check the knowledge base before starting external research.
- Always retrieve TaskOutput from previous steps before starting a new step.
- Always record architectural decisions in the Decision Log.

## ❌ NEVER Patterns
- Never execute unvetted code from untrusted sources.
- Never commit API keys, secrets, or PII into the repository.
- Never assume documentation is authoritative over the code.
- Never skip TaskOutput gates in sequential workflows.
